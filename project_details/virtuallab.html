Overview & Simulation Concept

Physical laboratory experiments are vital for engineering, but student access to high-precision hardware is often limited by scheduling constraints, equipment wear, and class sizes. To solve this, we set out to build an interactive, web-based simulation lab. The platform accurately replicates physical lab machinery, enabling students to learn and practice procedures virtually.

We utilized React to build a highly interactive front-end, rendering real-time measurements, dial rotations, and specimen projections. Controls mimic actual physical adjusters (knobs, switches, vernier readouts) to preserve the mechanical learning curve. In total, the system hosts 24 Metrology and Engineering Graphics experiments.

The Profile Projector Simulator

A profile projector - or optical comparator - is a key metrology instrument that magnifies a specimen's silhouette onto a screen, enabling precise 2D measurements. For our individual group project contribution, we built this complex simulator in React.

The simulator accurately models the physics of light path magnification, surface illumination, and x/y stage movement. We engineered custom React state handlers for micrometer dials, vernier scales, and digital readouts, ensuring that students can adjust focus, align grid lines, and measure specimen sizes down to the micrometer level.

My Role: Compiling, Structuring, & Gateway Design

The overall system was composed of 12 distinct student group projects (totaling 24 experiments). Originally, these projects were entirely decoupled, built as independent React modules with varying configurations. To prevent high server overhead and build a single unified college portal, I took the initiative to restructure and compile the codebase.

I structured the folders to support multi-app compilation, engineered a unified landing gateway dashboard, and mapped clean routes to each of the 12 decoupled React projects. This allows students to access any of the 24 experiments seamlessly from a single homepage link.

Institutional Publishing via XAMPP & NGINX

Once the compilation was complete, I configured the local infrastructure to host the platform on campus. I packaged the compiled static assets and configured the college's local XAMPP Apache servers to serve them.

To ensure efficient resource delivery, clean URLs, and routing isolation between the decoupled frontend modules, I configured an NGINX reverse proxy router. The proxy translates incoming campus traffic to the designated static endpoints, providing a seamless and secure virtual lab portal for all engineering departments.
