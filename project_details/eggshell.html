SQLite-Powered Local-First Engine

Traditional spreadsheet applications frequently lag, freeze, or completely crash when processing datasets exceeding tens of thousands of rows. To solve this problem, EggShell shifts the processing engine from basic memory arrays to a highly optimized database structure. The system instantiates an in-browser SQLite database running client-side inside a dedicated Web Worker.

By combining sqlocal and sql.js with the browser's Origin Private File System (OPFS), EggShell achieves near-native query execution times. Files are parsed, structured, and saved directly into private persistent storage. This architectural decision enables the application to execute complex analytical joins and filter queries across hundreds of thousands of rows with zero latency and complete visual responsiveness.

Visual DAG Pipeline & React Flow Canvas

Instead of writing nested Excel formulas or manual SQL code, developers and data analysts build pipelines visually using a Directed Acyclic Graph (DAG) editor powered by @xyflow/react (React Flow). Users import spreadsheets as source nodes and wire column handles through modular transformation blocks.

The canvas supports a variety of visual operators:

  • Transform Block: Quick text manipulations like converting strings to UPPERCASE or lowercase, trimming whitespace, and auto-generating serial row numbers.
  • Math Block: Build arithmetic structures (+, -, *, /) and trigonometric or logarithmic functions referencing numeric column headers.
  • Conditional Block: A visual builder to compile complex CASE WHEN ... THEN ... ELSE logic patterns.
  • Join Block: Links matching data tables using trimmed, case-insensitive LEFT JOIN operators.
  • Waypoint Node: Visual routing points to tidy up connector lines in complex workflows.

Grid Editor & Live Preview Monitor

To ensure that data is cleanly formatted before running compiler transformations, EggShell includes a secondary standalone grid editing viewport. This spreadsheet-like table editor enables developers to perform immediate database CRUD operations, append rows, prune records, rename columns, or reset the layout back to the original backup tables.

The workspace incorporates a live preview monitor. Whenever a connector handle is wired or an operator node properties drawer is adjusted, the visual SQL compiler automatically updates. The resulting SQLite query executes in the Web Worker, loading the updated dataset directly within the monitor viewport with custom column ordering, page limits, and multi-format export triggers.
