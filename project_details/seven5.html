Seven5 - Smart Attendance & Leave Simulation App

Timetable & Daily Schedule Dashboard

Managing class attendance at university requires strict tracking to meet the mandatory 75% attendance threshold. Seven5 simplifies this by providing students with an intuitive, dynamic scheduler interface. Users input their timetable with class period details (start and end times, teachers, and room numbers).

Once configured, Seven5's local scheduler engine dynamically parses the daily timetable and displays the current day's classes right on the home page. Users can log their attendance in real time—marking classes as Attended (green check), Bunked (red cross), or No Class—which instantly updates their local database and live attendance metrics.

Python Holidays Engine & Long Weekend Planner

To help students organize trips and vacations without hurting their academic standing, Seven5 integrates with a custom server API built on Python's robust holidays library.

The Holiday Planner module allows students to select their country and state or region (such as India and Tamil Nadu). The app queries the server backend to fetch region-specific official holiday lists, local observances, and weekends. These holiday arrays are cached locally in the app database, making the planner fully functional even in areas with poor internet connectivity. The backend then analyzes the academic calendar to locate holiday clusters, highlighting the best periods for long weekends.

Predictive Bunk & Leave Simulation

The core innovation of Seven5 is its predictive simulation logic. When planning a break, a student can select a specific timeframe (e.g. bridging holidays with planned leaves to take a 5-day holiday stretch). Seven5 runs a simulated calendar traversal: it parses the student's timetable for the target period, maps the classes they would miss, and projects the exact percentage drop for each subject.

If a planned leave will push a subject's attendance below 75%, the simulation flags a warning and calculates a safety strategy. It tells the student exactly how many classes of that subject they need to attend beforehand (e.g., "Attend English beforehand to keep your % safe") or warns them if it's statistically impossible, helping students make informed decisions.

Offline-First Sync & Google Play Publishing

Designed with an offline-first architecture, the application stores all schedules, timetables, and historical logs locally using Hive, a lightweight and blazing-fast NoSQL database for Flutter. To prevent data loss when switching devices, Seven5 implements a client-side Google Drive REST API integration, enabling encrypted cloud backup and restoring.

The backend service is containerized using Docker and deployed to Google Cloud Run, ensuring instant scaling and high availability. The application is currently in its deployment phase, with builds being pushed and certified on the Google Play Store console.
